# API Глоссария по Реактивному Программированию

## Описание проекта

Данный проект представляет собой RESTful API-сервис, предназначенный для хранения, поиска и управления терминами глоссария по теме "Исследование парадигмы реактивного программирования" в рамках Выпускной Квалификационной Работы (ВКР).

API предоставляет полный набор CRUD-операций (Create, Read, Update, Delete) для управления терминами.

## Используемые Технологии

- **Фреймворк:** [FastAPI](https://fastapi.tiangolo.com/ 'null').
- **Веб-сервер:** Uvicorn.
- **База данных:** SQLite с ORM [SQLModel](https://sqlmodel.tiangolo.com/ 'null') (на основе Pydantic и SQLAlchemy).
- **Контейнеризация:** Docker.
- **Язык:** Python 3.11+.

## Структура Проекта

```
.
├── Dockerfile           # Инструкции для сборки Docker-образа
├── requirements.txt     # Список зависимостей Python
└── app/                 # Основной пакет приложения
    ├── main.py          # Инициализация FastAPI и Lifespan-событий
    ├── database.py      # Настройка подключения к БД и сессий (SessionDep)
    ├── models.py        # Определения моделей SQLModel (GlossaryTerm)
    ├── crud.py          # Функции для работы с БД (CRUD-операции)
    └── api/
        └── glossary.py  # Роуты API для терминов (/terms)
```

## Развертывание с использованием Docker

Приложение полностью "контейнеризировано". База данных SQLite будет храниться внутри контейнера. При удалении контейнера данные будут утеряны.

### 1. Подготовка и Сборка Образа

```
docker build -t glossary-api .
```

### 2. Запуск Контейнера

Запустите контейнер, пробросив порт `8000` контейнера на порт `8000` хост-машины.

```
docker run -d \
    -p 8000:8000 \
    --name glossary-app \
    glossary-api
```

### 3. Доступ к API

После запуска контейнера API будет доступен по адресу:

- **Адрес API:** `http://localhost:8000`
- **Интерактивная документация (Swagger UI):** `http://localhost:8000/docs`

## Локальный Запуск

Этот метод требует установленного Python 3.11+ и используется в основном для разработки.

1. **Создайте и активируйте виртуальное окружение:**

```
python3 -m venv venv
source venv/bin/activate
```

2. **Установите зависимости:**

```
pip install -r requirements.txt
```

3. **Запустите Uvicorn-сервер:**

```
uvicorn app.main:app --reload
```

API будет доступен по адресу `http://127.0.0.1:8000/docs`.
